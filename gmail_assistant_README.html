Open WebUI Tool

Gmail Assistant

LLM-powered inbox triage — fetch, categorise, and flag emails for deletion using your local model.

v0.4 Python OAuth2 Gmail API Open WebUI

Prerequisites

Before installing the tool you need a Google Cloud project with Gmail API credentials and a mounted secrets folder in your Open WebUI Docker container.

1 — Google Cloud Setup

1
Create a project

Go to console.cloud.google.com → project dropdown → New Project. Name it anything (e.g. openwebui-gmail).

2
Enable Gmail API

APIs & Services → Library → search Gmail API → Enable.

3
Configure OAuth consent screen

APIs & Services → OAuth consent screen → External → Create. Fill in app name and your email. On the Test Users screen add your Gmail address. Save through all steps.

4
Create OAuth credentials

APIs & Services → Credentials → + Create Credentials → OAuth client ID → Desktop app. Download the JSON file and place it in your secrets folder:

mv ~/Downloads/client_secret_*.json ~/llm_local_gmail_creds/cred_llm_local.json
touch ~/llm_local_gmail_creds/cred_llm_token.json

2 — Docker Volume Mount

The secrets folder must be mounted into the Open WebUI container. Stop and recreate with the extra volume:

docker stop open-webui && docker rm open-webui

docker run -d \
  --name open-webui \
  -p 3000:8080 \
  -e WEBUI_AUTH=False \
  -v open-webui:/app/backend/data \
  -v $HOME/llm_local_gmail_creds:/llm_local_gmail_creds \
  --restart always \
  ghcr.io/open-webui/open-webui:main
Your conversation history and settings live in the named volume open-webui and are not affected by recreating the container.

3 — First-time Authentication

Run this once on your Mac to complete the Google OAuth flow and save the token:

/opt/homebrew/bin/python3.10 -m pip install google-auth-oauthlib google-auth-httplib2 google-api-python-client

cat > /tmp/gmail_auth.py << 'EOF'
from google_auth_oauthlib.flow import InstalledAppFlow
import os
flow = InstalledAppFlow.from_client_secrets_file(
    os.path.expanduser('~/llm_local_gmail_creds/cred_llm_local.json'),
    ['https://www.googleapis.com/auth/gmail.modify']
)
creds = flow.run_local_server(port=0)
open(os.path.expanduser('~/llm_local_gmail_creds/cred_llm_token.json'),'w').write(creds.to_json())
print('Token saved.')
EOF

/opt/homebrew/bin/python3.10 /tmp/gmail_auth.py

A browser window opens — log in with Google and approve access. The token is saved to the mounted folder and is immediately visible inside Docker. You will not need to do this again — the tool refreshes the token automatically.

4 — Install in Open WebUI

1
Open Open WebUI → Workspace → Tools → + New Tool
2
Paste the full contents of gmail_assistant.py and click Save. Dependencies install automatically.
3
Create a model — Workspace → Models → + New Model named Email Assistant, base model qwen3-4b, tools: Gmail Assistant enabled, system prompt below.

System Prompt

You are an email triage assistant. When asked to process emails:
1. Call get_processing_options() and present the two choices to the user
2. Wait for the user to choose, then call fetch_unprocessed_emails(mode)
3. Call categorise_email() for EACH email using its EXACT id — never placeholders
4. For emails where auto_to_delete is true, always set to_delete=true
5. After each batch fetch the next if more_remaining is true
6. When done, call get_summary() and present the results
7. Never call fetch_unprocessed_emails() more than 20 times in one session

5 — Persistent Files

~/llm_local_gmail_creds/
├── cred_llm_local.json ← OAuth credentials from Google Cloud
├── cred_llm_token.json ← OAuth token (auto-refreshed)
├── state.json ← last run timestamp + total count
└── email_log.md ← full history of all decisions

All files live on your Mac and survive Docker container restarts or rebuilds.

Usage

Select the Email Assistant model in Open WebUI and send:

Process my emails
fn
get_processing_options()
Returns last run date and email counts for both modes
↓
Model asks: resume from last run (N emails) or last 24h only (N emails)?
You choose
fn
fetch_unprocessed_emails(mode)
Returns batches of 5 emails with auto_to_delete hints
fn
categorise_email() × 5
Applies Gmail labels, appends one line to email_log.md
fn
get_summary()
Reads the log and presents the final triage report

Available Functions

FunctionDescription

fetch_unprocessed_emails(mode)Fetch next batch. mode: last_24h or since_last_run

get_summary()Return session log for final report
